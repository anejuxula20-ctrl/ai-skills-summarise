## 结论速览

1. 本轮 GitHub AI 热度仍被三类主题主导：AI agent、AI coding assistant、AI skills。Trendshift 的日榜把这三类放在前三，其中 AI agent 主题日榜汇总约 24.5k stars，AI coding assistant 约 9.3k，AI skills 约 8.6k。
2. 过去一周增长最快的项目不全是传统 agent 框架，更多是“agent 能力包”和“内容生产/理解工具”：MoneyPrinterTurbo、Understand-Anything、Hermes Agent、ECC、Andrej Karpathy skills、guizang-social-card-skill 都在榜。
3. “技能包”生态开始出现安全需求。NVIDIA/SkillSpector 这种扫描 agent skills 风险的工具进入高增长列表，说明安装第三方技能不再只是效率问题，也变成供应链安全问题。
4. star velocity 只能代表开发者注意力，不等于生产采用。MoClaw 和 TechTarget 都提醒应把 star 增长当作早期信号，而不是选型唯一依据。

## 技术趋势判断

1. Agent 生态从“单体自主代理”转向“可插拔能力栈”。Hermes、ECC、skills 类仓库的快速增长说明开发者正在把 agent 当作可扩展运行时，而不是一个单一聊天机器人。
2. 内容生产类 AI 工具仍然非常吸星。MoneyPrinterTurbo 和 guizang-social-card-skill 都把 AI 直接连接到可发布产物，前者偏视频，后者偏社媒图文。
3. 代码理解正在变成 agent 的核心基础设施。Understand-Anything 的增长说明“让 agent 理解大型代码库/知识库”是强需求，图谱化与可问答是当前最容易传播的表达方式。
4. 语音 agent 进入开源自建栈阶段。Patter 的关注点不是再包装一个 SaaS，而是把电话、实时语音、LLM、STT/TTS、观测性统一成开发者可控的 SDK。
5. 技能安全会成为新入口。SkillSpector 的出现很像早期 npm/pip 安全扫描，区别是 agent skills 可能直接影响模型行为、文件操作、命令执行和外部服务访问。

## 风险与不确定性

1. stars 可能被营销、榜单曝光或 fake-star 行为扭曲，尤其是新 repo 的短期百分比增长。
2. 一些高星项目是文档、索引、技能文件或提示词集合，不等于可运行基础设施。
3. 内容生成项目要额外审查版权、素材来源、模型调用成本和平台风控。
4. agent skill 的安装信任边界还不成熟，建议在自动化环境中增加白名单、静态扫描和沙箱运行。
5. 榜单来源之间口径不同：AI Signal、Trendshift、TechTarget、GitHub topic 页分别使用自己的采集窗口与排序算法，本报告把它们作为趋势交叉验证，不把某一个榜单当作绝对事实。

## 下次自动化建议

1. 若要真正“发布到 GitHub”，请给当前仓库添加远程，例如 `git remote add origin <your-repo-url>`，或告诉自动化目标仓库。
2. 自动化可以在报告里固定保留“增长数据来源”和“实时 GitHub stars”两列，避免后续读数更新造成歧义。
